Web Publishing:


How to choose the right host for you?

There are 3 things you need to look out for when buying a host


1: Latency: if hosting a site for local use only, use a host with servers relatively close, if hosting site globally, use a host with servers across the world


2: Cost: Don't buy a host that is too expensive for your needs, only get what you need. However the inverse works as well, don't underpay for hosting


3: Service Quality: Don't buy the bottom of the barrel host unless you can work around the limitations that may be provided, such as high downtime

Search Engine Optimization

How to get your website to show up on google

These are 5 ways to improve your website on google

1: Use a Robots.txt file: Robots.txt tells google where on your site not to trawl for search results, useful for say, a shopping cart


2: Use Title tags correctly: Title tags will show up as the pages name on google, so name it well


3: Use Meta tags correctly: the description tag will be used for the sites description on google


4: Use Structured data markup: this is more useful for stores, but if you use structured data markup, it will allow google to display prices and reviews for whatever product it is


5: Keep your URLs simple: keeping your urls simple allows for google to parse it easier

Web Security

How to prevent your systems from getting PWNed

Here is 5 common attacks and how to prevent them

1: Man in the middle attack: man in the middle attack is when a malicious actor hijacks the connection between a server and user, generally by being on the same network as the user

How to prevent?: Use HTTPS. HTTPS encrypts traffic, preventing this from happening

2: Attacks using legacy attack vectors: Currently most famous in New Zealand for the DHB hack, legacy attack vectors rely on software that has not been updated but security flaws have been published for it

How to prevent?: Update software when it becomes EOL(End Of Life)

3: DDoS: Distributed Denial of Service, or DDoS is when a server is overloaded with requests, generally from malicious people, using botfarms

How to prevent?: Use a DDoS mitigation service such as Cloudflare

4: Breaches in closed source software: Closed source software does not allow 3rd parties to verify security as the code is hidden from the user, this can be used if someone finds a vulnerability

How to prevent?: Use open source alternatives to closed source software, as open source allows you to look at the code and verify for yourself, alongside others

5:Accidentally leaving ports open: ports allow for programs to communicate over the internet, however some can do more than others, such as port 1433

How to prevent?: close all ports except for those that are necessary

Creative Commons and Copyright

Here i will discuss the basic principles of Copyright, 4 types of Creative Commons licenses and Fair use

Copyright is a system that protects authors works, such as an artists work, or a writers books, it lasts for the authors lifetime plus 50 years

Attribution-NoDerivativess (CC BY-ND)

this license lets people reuse the work as long as it is not altered and as long as they credit you

Attribution-NonCommercial (CC BY-NC)

this license does not allow for it to be used commercially, and it does not have to be licensed under the same terms if adapted

Attribution-NonCommercial-ShareAlike (CC BY-NC-SA)

this license does not allow commercial use, must be credited, and has to have the same terms even if adapted

Attribution-NonCommercial-NoDerivs (CC BY-NC-ND)

this license does not allow commercial use, must be credited, has to have the same terms, and cannot be edited

Web performance and maintenance

Here i will discuss 5 ways to speed up page loading time and 5 tasks to carry out when maintaining a website

Loading times

1: put javascript at the bottom of the page, to allow for it to load last

2: use appropriately sized images, reducing bandwidth used loading it

3: use a high performance host

4: reduce use of redirects

5: use a CDN(Content Delivery Network) to allow the server hosting the webpage to be closer to the user
